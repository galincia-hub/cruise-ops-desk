# 코스타 세레나 전세선 운영 — cruise-ops-desk

## 토큰 효율 규칙
- 이미 읽은 파일은 같은 세션에서 다시 읽지 않는다
- 불필요한 도구 호출을 하지 않는다 (확인용 재조회 금지)
- 병렬 실행 가능한 도구 호출은 동시에 실행한다
- 20줄 이상의 반복 출력은 서브에이전트로 위임한다
- 사용자가 이미 설명한 내용을 반복하지 않는다

## 프로젝트 개요
- 코스타 세레나 한일전세선 (2026.6.19~25, 부산→하코다테→오타루→부산)
- 6박7일, 약 2,400명, Costa Serena 114,000톤
- 공동운영: 모두투어(주관) + 환경재단 + 웅진프리드
- 웹사이트: https://galincia-hub.github.io/cruise-ops-desk/
- GitHub: galincia-hub/cruise-ops-desk

## 시스템 역할
- 운영 대시보드 + 팀별 워크스페이스 + 참고자료실 웹사이트 관리
- 엑셀 → JSON 변환 → 웹사이트 반영 파이프라인
- 마스터 업무 매트릭스 119건 관리 (시간축 10개, 카테고리 16개)

## 조직 표기 규칙 (필수)
- "공동부서장" 표현 사용 안 함 → "HQ (모두투어 + CI)"
- 모두투어와 CI의 역할을 구분하지 않음 → 통합 표기
- 모두투어를 먼저 기재, CI를 다음에 기재
- 황지애: "CI원" 표기 금지 → "CI", 팀장 아님 → 지원/백업 역할
- 최구철: IT홍보팀 "팀장", "파견" 표시 안 함
- 외부 실명 제거

## 핵심 용어
- 듀얼: 타임스케줄별 복수임무자 (≠듀프)
- 사전탑승자: 직전 항차(6/13~19)에 미리 승선하여 세팅하는 인원
- 베이스캠프: 승선 전날(6/18) 부산 운영 거점
- SOP: 표준 운영 절차서

## 에스컬레이션
- 금전/예약/취소 관련 → 절대 자동화 금지
- data 폴더 ZIP 교체 → 한 번에 모아서 처리 (여러 번 교체 시 충돌 위험)
- 조직도/인사 변경 → 사람 확인 필수
